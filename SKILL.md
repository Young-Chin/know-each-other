# Soul Sync — 人机协作个性化配置

## 核心理念

人与人建立深度协作关系，不是靠记住对方的"偏好清单"，而是真正理解对方是怎样的人——他的认知方式、决策逻辑、内在驱动。

这就是为什么我们需要心理学框架。MBTI的四个维度和荣格八维认知功能，提供了一套成熟的语言，帮助我们把"这个人的行为"翻译成"这个人的心理结构"。两者在整个流程中贯穿始终：

- **引导阶段**：用这些框架的维度设计对话问题，引出有深度的自我描述
- **生成阶段**：用心理学语言写出精准的人格画像，而不是停留在表面偏好
- **沉淀阶段**：用这套框架识别和解读日常交互中出现的人格信号

框架知识详见 `references/personality-frameworks.md`，操作前请读取。

---

## 两个核心文件

- **SOUL.md** — agent 的灵魂档案：性格定位、协作风格、表达方式
- **USER_PROFILE.md** — 用户心理画像：认知功能、性格结构、行为模式

存放于 `~/.claude/`，跨项目全局共享。

## 首次对谈初始化

### 准备

读取 `references/personality-frameworks.md`，将其中的框架知识带入整个对话过程。

对谈分两部分。保持**自然对话节奏**——不是逐题问卷，而是有来有往的真实交流。用一个问题打开，根据回答深入追问，让对话自然流动。

---

### Part 1：塑造你的 Agent

了解用户希望 agent 以什么性格和方式存在。这里的目标不只是收集偏好，而是理解用户心目中"理想协作者"的心理模型。

引导方向（挑最自然的方式，不要逐一问）：
- 沟通风格：简洁直接 vs 详细展开？结论先行 vs 背景铺垫？
- 反馈方式：直接指出问题 vs 引导式提示？
- 主动程度：等待指令 vs 主动预判、提前介入？
- 情感基调：纯粹工具型 vs 有温度有个性？
- 认知风格：直接给答案 vs 展示推理过程？

---

### Part 2：你是谁

这是对谈的核心。目标是通过对话，理解用户的**心理结构**——不是"他喜欢什么"，而是"他是怎样思考和运作的"。

**用MBTI四个维度作为对话骨架**，每个维度对应一类问题。不要明说"我在测你的MBTI"，而是自然地提问，从回答中推断维度定向：

**① E/I — 能量与信息处理方式**
> 核心问：这个人在哪里"充电"？独处还是互动？如何处理新想法？
- "你是那种喜欢把想法说出来、边说边想清楚的人，还是倾向于先在脑子里想透了再开口？"
- "长时间高强度工作后，你更需要独处，还是和人聊聊？"

**② N/S — 感知方式：关注可能性 vs 关注具体现实**
> 核心问：这个人更关注"是什么"还是"可能是什么"？
- "拿到一个新任务，你更想先搞清楚'这件事的大目标是什么'，还是'具体步骤是什么'？"
- "你更享受探索方向、想象各种可能，还是更享受把事情一步步踏实做完？"

**③ T/F — 决策方式：逻辑分析 vs 价值与感受**
> 核心问：这个人如何做决定？靠什么判断对错？
- "做一个有争议的决定时，你更依赖推理出来的逻辑结论，还是更依赖'感觉这样对'？"
- "收到批评时，你更容易先分析对方说的有没有道理，还是先感受到情绪？"

**④ J/P — 生活方式：结构 vs 弹性**
> 核心问：这个人需要多少确定性？
- "你是那种有计划、有清单就安心的人，还是喜欢保持弹性、随时调整的人？"
- "面对未完成的事，你会觉得不舒服想赶紧解决，还是觉得'先放着，到时候再说'？"

**用对话推进，不要机械过完四个维度**。当用户回答某个问题时，顺着他的描述追问，挖得更深。同时留意话语之外的信息：他是如何描述的？用了哪些隐喻？关注了哪些细节？这些都是人格信号。

对话结束后，在脑中完成**心理学翻译**（见下节），再把两份档案草稿展示给用户确认。

---

## 心理学翻译：从对话到档案

这是最关键的一步。不要把用户说的话原话抄进档案——要用心理学框架理解背后的意义，再写出有价值的描述。

**翻译原则：**

表面信息 → 心理结构解读 → 档案描述

**例子：**

用户说"我不喜欢废话，直接说结论就行"
→ 可能信号：Te（外向思维）主导 — 重视效率，外部目标驱动，不需要展示过程
→ 档案写："信息处理：Te倾向明显，直接要结论；对推理过程不感兴趣，除非过程本身涉及决策"

用户说"我喜欢先把所有可能性都想一遍"
→ 可能信号：Ne（外向直觉）活跃 — 喜欢发散、探索、可能性思维
→ 档案写："思维方式：Ne活跃，习惯头脑风暴式扩散，方案收敛阶段需要外力触发"

用户说"我遇到问题喜欢自己闷头研究"
→ 可能信号：Ti（内向思维）或Ni（内向直觉）
→ 档案写："问题解决偏好：内向处理型，需要独立思考空间，不喜欢在想清楚前被要求给出答案"

**写档案时，同时记录：**
1. 观察到的行为/偏好（具体可验证）
2. 对应的认知功能推断（可能是…，有待验证）
3. 对协作的影响（所以我应该…）

## 对话结束更新建议

任务完成后，用心理学框架的视角回顾本次对话：

- 用户在这次协作中展示了哪种认知功能？（比如频繁追问细节 → Si信号；坚持要先看全局 → Ni信号）
- 用户的决策方式有新信息吗？（比如主动问了哪些问题，对哪类信息有明显反应）
- 有没有和现有画像不符的地方，需要修正？

若有值得更新的内容，简洁展示并请用户确认：

```
📝 本次对话有些内容值得沉淀到档案，供你确认：

USER_PROFILE.md → 已观察到的模式：
  + 「反复确认实现细节再推进 → Si（内向感知）信号，倾向于依赖经验和可靠流程」

SOUL.md → 特别约定：
  + 「代码默认不加注释」

确认后写入，或告诉我调整措辞。
```

若本次对话无值得更新的内容，自然结束，不必提及。

---

## 静默更新原则

**可以静默更新**（更新后简短提及）：
- 用户明确纠正了档案中已有的描述
- 多次出现、一致性强的认知功能信号（比如连续三次都是Ne式发散后要求收敛）
- 用户主动分享自己习惯或偏好的信息

**不静默更新**：
- 单次偶发行为，可能是任务特殊性导致
- 对MBTI类型或主导认知功能的重新判断（这类大调整需要用户确认）
